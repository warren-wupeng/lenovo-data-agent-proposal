Roadmap

Three-Phase Implementation

A pragmatic, validation-driven approach: start with what works, learn from real usage, scale only when the data justifies it.

1

Genie Customization & Enhancement

Seed Team + Deep Semantic Annotation

Build on Databricks Genie's existing Text-to-SQL capability. No reinventing the wheel — focus on domain-specific depth.

  • Select 1-2 seed teams for pilot deployment
  • Annotate top 50 high-frequency query scenarios with business semantics: field definitions, metric calibers, common ambiguities
  • Implement confidence scoring — low-confidence results actively prompt user confirmation
  • Build semantic layer: map business terms to table schemas and column-level lineage
2

Memory & Feedback Loop

From Coding Agent V4: Conversation Compression

The Agent learns from corrections and never repeats the same mistake. Directly transferring proven techniques from production coding agent.

  • User correction memory — Agent records and avoids repeating errors
  • Conversation compression engine: auto-trigger at 25-message threshold, 40-60% token reduction (measured in production V4)
  • Query pattern caching — frequently asked questions get instant structured responses
  • Cross-session memory: business context persists across conversations
3

Demand-Driven Multi-Agent

Supervisor + Specialized Agents

Scale architecture only when real usage patterns justify it. Not architecture-first, but demand-first.

  • Supervisor Agent orchestrates task routing
  • Structured Data Agent — SQL generation, aggregation, trend analysis
  • Document Agent — internal docs, policies, unstructured knowledge retrieval
  • Sub-agent delegation: 60% → 85% success rate on complex queries (proven in coding agent V4)

Methodology

Technical Approach

Insights from building 5 versions of a production coding agent — 33 lines to 9,266 lines — applied to data analysis.

Layered Context Architecture

The hardest problem in data analysis agents is not SQL generation — it is context. "What were last month's shipments?" means different things to different departments.

Inspired by OpenAI's internal data agent practice (serving 3,500 people across 600PB of data), we build context in layers:

  • Layer 1: Global schema — table structures, column types, relationships
  • Layer 2: Domain semantics — business definitions, metric calibers, department-specific terms
  • Layer 3: User context — role, team, query history, correction history

Core challenge: fitting the most valuable context into a limited token window. Same problem, different domain.

Lessons from 5 Agent Versions

Building a coding agent from scratch taught us what works and what doesn't when taking agents from prototype to production:

  • V0-V2: Proved feasibility. LLM + tool calling works, but fails on complex tasks.
  • V3: Architecture inflection point. Controller/Action pattern enables extensibility.
  • V4: Production breakthrough. Sub-agent delegation (60%→85% success), conversation compression (40-60% token savings).
  • V5: Quantitative evaluation. SWE-bench framework to objectively measure strategy effectiveness.

The biggest leap was never a better model — it was better context management and tool orchestration.
